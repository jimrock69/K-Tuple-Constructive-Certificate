## 🗺️ Core Architecture: The Primorial Window

To look for dense constellations like octuplets ($D_8$) or decuplets ($D_{10}$), the search space is restricted to coordinates generated by the primorial arithmetic progression:

$$C(n) = 105 + 210n$$

Where $M_4 = 2 \cdot 3 \cdot 5 \cdot 7 = 210$. This geometric framework guarantees a $0\%$ filtration rate for the opening phases of the sieve:
* **$\pmod 2$:** $C(n)$ is identically odd; all even offsets are preserved.
* **$\pmod 3$:** $C(n) \equiv 0 \pmod 3$; no offset in the target constellation is divisible by 3.
* **$\pmod 5$:** $C(n) \equiv 0 \pmod 5$; no offset in the target constellation is divisible by 5.
* **$\pmod 7$:** $C(n) \equiv 0 \pmod 7$; no offset in the target constellation is divisible by 7.

This reduces the effective sieve demand $k_{eff}(P_n)$ to zero during the critical opening stages of the expansion, creating a massive local candidate surplus.
